# Assumptions & Scope

## Prerequisites
- Basic kinematics & vectors \((x,y,\psi,v)\)
- Curvature \(\kappa\) on a 2D path; tighter bend ⇒ lower feasible speed
- PID intuition (proportional control, gain scheduling)
- Simple ODE stepping (Euler/RK)

## What we assume we already have
- Reference path \(\gamma(s)\) with curvature \(\kappa(s)\) (centerline from cones)
- State estimate \((x,y,\psi,v)\) each control tick
```math
- Grip estimate \(\mu\) and actuator limits \((\delta_{\max}, \dot\delta_{\max}, a_{x,\max})\)
```
- Lightweight browser simulator (kinematic bicycle + friction-limited target speed)
- Three tracks: **S-bend**, **Hairpin**, **Chicane**

> Perception is out of scope. We assume a clean path and usable state so we can focus on control.

## Design choices (intentional)
- Kinematic bicycle for **legibility** and speed of iteration
- Interpretable **sliders** over weight matrices (predict before running)
- Preset tracks & ghosts to isolate **style** from randomness
- Metrics chosen for teaching: lap time, lat-g, mean jerk, apex-miss, overshoot

<Separator className="my-10" />

## Guarantees & limitations

**We guarantee**
- Stable sims, deterministic replays
- Parameter changes produce **predictable** visual differences

**We do not guarantee**
- Absolute lap-time realism or tyre-level fidelity
- Direct transfer to a specific real vehicle

**Trade-off:** interpretability and iteration speed over raw optimality.

<Separator className="my-10" />

## Evaluation policy

- **Primary:** can you **predict** the trajectory/metrics from the style settings?
- **Secondary:** metrics vs baselines (lap time, jerk, apex adherence)
- **Tertiary:** robustness under small \(\mu\) and lookahead \(L\) perturbations

## Reproducibility & knobs

- Seed-fixed demos: same start, same \(\mu\), same \(L\)
- **Sliders:** Apex offset, Entry/Exit bias, Risk, Smoothness
- **Toggles:** grip \(\mu\), lookahead \(L\), Baseline vs MPCC-like ghost
- **Overlays:** curvature heatmap, speed band, friction “donut”, error traces
